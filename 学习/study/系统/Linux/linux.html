技巧：

文件目录操作：/ 为根目录；~ 为家目录；. 为当前目录；.. 为上一层目录；- 为上一次操作的目录

文件类型：- 为普通文件；d 为目录；c 为字符设备；b 为块设备；s 为socket；l 为软链接

操作权限：r 为可读；w 为可写；x 为可执行；其中可用数值表示，421 其中 4 为读 2 为写 1 为执行，+-= 为增加减少设置权限

通配符：
星号*：匹配任意长度字符串
问号?：匹配一个长度字符
方括号[...]：匹配其中指定的一个字符
[-]：匹配指定的一个字符范围
[^...]：处理其中指定字符均可匹配

管道：将一系列命令连起来，以“|”作为管道连接符。第一个输出作为第二个输入，以此类推。
如：ls /usr/include/ | grep std，管道过滤，即将当前目录下的 ls 获取到的数据通过管道到 | 右边，然后右边使用 grep std 获取 ls 之后中有 std 字符的文件显示出来，比如 stdio.h unistd.h1等形式。

重定向：输入输出重定向，改变 shell 命令或程序默认的标准输入输出目标，重新定向到新目标。
> file：将文件重定向文输出源，新建模式
>> file：将文件重定向为输出源，追加模式
< file：将文件重定向为输入源
> 将命令产生的错误信息输入到重定向的指定
如 ls > test.txt 将 ls 命令输出的结果输入到 test.c 文件，若此文件已存在则会将原来的数据也覆盖，可使用 >>追加到后面，这样就不会覆盖了。

软硬链接的区别：
硬链接：1.给某个文件拷贝一份这两个文件独立占用内存；2.改变一个文件会影响另一个文件；3.删除某个硬链接文件不会影响另一个文件。
软链接：1.类似于 window 下的快捷方式，软链接依赖于源文件存在；如果源文件已删除则软链接也将不可用。

/etc/group：如 edu:x:1001:1000:edu,,,:/home/edu;/bin/bash：用户名：edu，用户id：1001，用户组：1000

IO编程：1.标准IO；2.文件IO
标准IO：1.接口fopenfread/fwrite/fclose；2.ANSI C标准（可移植性好）；3.缓冲；
  缓存：1.目的是尽量减少使用read/write的调用； 2.定义为系统自动的在内存中为每一个正在使用的文件开辟一个缓冲区，从内存向磁盘输出数据必须先送到内存缓冲区，装满缓冲区在一起送到磁盘中去。从磁盘中读数据，则一次从磁盘文件将一批数据读入到内存缓冲区中，然后再从缓冲区逐个的将数据送到程序的数据区。3.缓冲类型
    缓冲类型：1.全缓冲；2.行缓存；3.无缓冲；
    全缓冲：1.冲刷机制为缓冲区满或关闭设备后进行缓冲区冲刷；2.强制冲刷接口fflush；3.容易遇到问题即掉电数据丢失或文件损坏；
    行缓存：冲刷机制为当在输入和输出中遇到新行符 \n 时，进行I/O操作；容易遇到问题时加了打印无输出，忘加 \n
    无缓冲：冲刷机制为立即冲刷；应用场景为标准出错
文件IO：1.接口open/read/write/close/ioctl；2.要点；3.无缓冲且实时性高，对字符设备兼容性好，但效率低；
  要点：1.对内核而言所有打开文件（包括设备）都由文件描述符引用；2.文件描述符是一个非负整数，当打开一个现存文件或创建一个新文件时，内核向进程返回一个文件描述符；3.在posix.1应用程序中，幻数0、1、2应被代换成符号常数STDIN_FILENO、STDOUT_FILENO、STDERR_FILENO

网络编程：1.基础；2.流式套接字；3.数据包套接字；4.服务器模型
  基础：1.网络模型为 数据包 -> FTP头+数据包[应用层数据包] -> TCP头+FTP头+数据包[TCP段] -> IP头+TCP头+FTP头+数据包[IP数据报] -> 以太网头(令牌网头，这是解析端主机)+IP头+TCP头+FTP头+数据包+CRC[以太网数据帧/令牌环网数据帧（解析端主机）]，箭头方向的为发送端数据封装过程，反向箭头为接收端数据解封过程；路由器接收到以太网数据帧并根据目标IP地址转发到令牌环网接口（目标主机）；2.ip地址为inet_aton()/inet_addr()/inet_ntoa()；3.端口号：字节序/ntohs/ntohl；4.三次握手四次挥手；5.wireshark抓包分析
  流式套接字(tcp)：1.客户端socket()/connect()/write()/read()/close()；2.服务器端：socket()/bind()/listen()/accept()/read()/write()/close()
  数据包套接字(udp)：1.客户端：socket()/sendto()/recvfrom()/close()；2.服务器端：socket()/bind()/recvfrom()/sendto()
  服务器模型：多进程模型、多线程模型、select模型、epoll模型

进程线程：1.进程；2.线程
进程（资源分配的最小单位）：1.概念也可以称为任务，指的是一个软件或者一个可执行程序的一次运行。可执行程序：静态，放到磁盘，等待执行；进程：动态，占有内存资源，CPU发生调度。2.特性；3相关接口；4.进程间通信
  特性：1.并发性（同一时间可运行多个进程）；2.动态性（进程的状态不断变化）；3.交互性（允许过程中会和其他进程发生直接或间接通信）；4.独立性（拥有完整的进程资源，地址空间独立）
  相关接口：1.fork（创建子进程，父进程空间独立）；2.exec函数族（提供在进程中执行另一个程序的方法，另一个程序可为已存在的二进制或脚本程序）；3.system（执行系统命令）；4.exit（进程退出）；5.wait（等待子进程）
 进程间通信：1.无名管道；2.有名管道；3.信号；4.信号量；5.消息队列；6.共享内存；7.unix套接字（特点为复用socket接口，通信效率高，数据量大）
    无名管道：1.只能用于有亲缘关系的进程之间的通信，单工通信，管道其实是一种特殊的文件描述符，可直接进行读写，并不属于文件系统，存在内存；2.接口pipe()/popen()
    有名管道：1.特点为支持两个不相干进程实现彼此通信，管道可以通过路径名来指出，文件系统可见，建立后当做普通文件读写，允许双向通信，但同一时间只允许半双工通信，若想双向通信需开两个管道；2.接口mkfifo
    信号：1.特点类似硬件中断，是软件层对中断机制的一种模拟，是异步的，事件不一定何时发生，进程无需等待；2.进程对信号的响应方式为忽略信号(SIG_IGN)，除SIGKILL和SIGSTOP外，执行相应的处理函数，执行linux默认操作(SIG_DFL)；2.接口sighandler_t/signal()/pause()/kill()/alarm()
    信号量：1.特点为保护共享资源，使得资源在一个时刻只有获得信号量的进程（线程）所拥有；2.接口sem_init()/sem_wait()/sem_trywait()/post()
    消息队列：1.特点类似FIFO先进先出，拥有消息类型，可以实现消息的随机查询；2.接口ftok()/msgget()/msgrcv()/msgsnd()
    特点为通过将一段内存映射到用户进程空间从而达到进程间的通信方式，比起其他通信方式，共享内存是进程间共享数据的一种最快的方式；2.注意点为多进程之间对一个给定存储区访问的互斥；3.接口shmget()/shmat()
线程：1.接口pthread_create()/pthread_exit()/pthread_join()；2.互斥pthread_mutex_lock()/pthread_mutex_trylock()/pthread_mutex_unlock()；3.同步pthread_cond_wait()/pthread_cond_broadcast()/pthread_cond_signal()；4.线程与进程区别
  线程与进程区别：1.进程是操作系统资源分配的基本单位，而线程是任务调度和执行的基本单位；2.每个进程都有独立的代码和数据空间，程序之间的切换会有较大的开销；线程可以看做轻量级的进程，同一类线程共享代码和数据空间，每个线程都有自己独立的运行栈，线程之间切换的开销小；3.在操作系统中能同时运行多个进程（程序）；而在同一个进程（程序）中有多个线程同时执行（通过CPU调度，在每个时间片中只有一个线程执行）

GCC小知识
GCC编译流程：编译器编译.c源代码 -> 预处理器成.i预处理文件 -> 编译器成.s汇编文件 -> 汇编处理成.o汇编文件 -> 链接成.exe可执行文件，即预处理、编译、汇编、链接
GCC基础用法：gcc [options] [filenames]；-c 只编译不链接成可执行，生成.o目标文件；-o 生成可执行文件，若没有给出则默认的可执行文件为a.out，如gcc –o test test.c；-g 产生符号调试工具（GNU的gdb）所必须的符号资讯，对源代码进行调试时必须加入此选项；-O 对程序进行优化编译、链接，编译速度慢但是产生的可执行文件执行效率可以提高；-O2 比-O更好的优化编译、链接，但编译和链接速度更慢；-ldirname 将dirname所指向的目录加到程序头文件目录列表中，在预处理过程中使用；-Ldirname 将dirname所指向的目录加入到程序函数档案库文件的目录列表中，是在链接过程中使用的参数
GCC相关操作详情请看Makefile

以下为命令行具体使用方式

  • shutdown -h now：现在就关机
  • sync：用于数据同步，在关闭linux系统时使用。若直接写入，系统突然断电时数据会丢失，而此指令将存于buffer的数据强制写入硬盘。
  • man：接口帮助文档
    [] 为可选，<> 为必选，ctrl+f 向下翻页，ctrl+b 向上翻页，q 退出，/ 查找，... 为可变参数。如 man ls
  • su：切换用户
  • exit：退出当前用户，切换到上一用户，若为图形界面则退出当前终端，若是ssh远程登陆则退出登录账户，若是切换后的登陆用户则返回上一个登录账号
  • sudo：暂时以管理员权限执行
  • cd：切换目录
    cd ~：转到 /home/user 用户目录下
    cd /：转到根目录下
    cd /xxx：绝对路径，转到 xxx 目录下
    cd xxx：相对路径，转到当前目录下的 xxx 目录
  • ls：查看当前文件夹内容，-a 显示隐藏文件（即以.开头文件），-l 显示具体信息，windows 下是 dir。
  • pwd：显示当前文件夹绝对路径
  • cat：显示文件内容，可同时显示多个文件，如 cat 1.txt 2.txt
  • more：分屏显示文件内容
  • mkdir：创建目录，-p 选项可进行递归创建
  • rmdir：删除空目录，目录里面不能有其他目录或文件。递归删除可用 rm 命令
  • touch：创建普通文件，若已经有此文件则会更改文件的创建时间，如 touch *.c 是将当前目录下的文件的创建时间都改成当前时间，若是想创建 *.c 则直接使用 touch \*.c
  • rm：删除文件或目录，-r 删除目录，-f 不询问
  • cp：复制，-r 拷贝目录，-i 有提示，-s 没提示，-v 有进度
  • mv：移动，-v 有进度，-i 有提示，-s 没提示
  • chmod：修改文件或目录权限
  • tar：只负责打包盒解包，不进行压缩，压缩解压。-c 创建压缩包，-v 进度，-f 文件，-x 解压
    打包 tar -cvf xx.tar，解包 tar -xvf xx.tar（默认解压到当前路径），注意解包时若需要加路径则需要选项 -C 。
    tar.gz 解压为 tar -zxvf xxx.tar.gz，gz 解压为 gzip -d xxx.gz。
    若报错 tar: Error is not recoverable: exiting now：1.减去选项 -z 即可，即 -vxf；2.若还是不行则使用 unzip 命令（需要嵌入插件，可用yum云下载，yum install -y unzip zip）；3.可能是因为在共享文件夹下解压的，换其他位置试试
  • find：查找文件，在硬盘或磁盘查找某个文件，-size 查找大于或小于某个值的文件，如 find -size +1M 大于一兆的文件
  • grep：在文件内查找字符串，grep make 1.txt，在 1.txt 文件里查找 make 字符，i 选项忽略大小写，-n 显示行号，-v 取反（凡是没有 make 字符的都显示出来）
    可配合通配符一起使用，如 grep "1 2 3" 1.txt，这里查询的是字符“1 2 3”，若不加双引号或单引号则只会查找 1 这一个字符。
  • vi：命令模式，编辑模式（按i进入，按esc退出），底行模式（命令模式下按shift+:）
  • ps：列出进程
  • ping：测试网络连通性
  • ipconfig/ifconfig：查看网卡信息
  • --help 选项查看对应命令的相关使用方式，如 ls --help，
  • history：显示前面已经敲过的命令
  • clear：清屏（windows 下的是 cls）
  • ln：创建链接， 不加参数表示创建硬链接，-s 表示创建软链接，如 ln test.txt 1.txt 把原文件 test.txt 创建一个硬链接 1.rxr，而 ln -s test.txt 2.txt，即创建 test.txt文件的软链接。硬链接相当于两个文件，改一个另一个也会改，并且硬链接只能是文件不能是目录，而软链接不占内存，源文件没有了则软链接也会打不开。
  • file：查看文件类型，如二进制文件或文本文件
  • du：查看内存实际大小，比 ls -lh 只是查看目录大小，而不是目录全部内容
  • while：查看命令的位置
  • whoami：查看当前用户
  • who：查看当前用户，比 whoami 更详细
  • w：查看当前用户，比 who 更详细
  • groupadd：增加用户组
  • groupdel：删除用户组
  • uermode：修改用户所在组，如 usermode -g xx xxx，将 xx 用户加入到 xxx 组
  • apt-get install：安装软件

  • 小知识（或报错）：
    1. VMware workstation player 下的 VMware Tools 呈现灰色：
      在虚拟机中点到对应的系统上，然后找到“虚拟机设置”，在这里面的“硬件”选项里面的“CD/DVD”、"CD/DVD 2"、软盘这三个都设置成自动检测（即选中 使用物理驱动->自动检测），设置完后点击确定，之后再启动系统，这是会就不会呈现灰色，可以点击Vmware Tools了
    2. VMware 虚拟机的 linux 实现和 window 主机做目录映射实现：

      VMware 共享实现：
        1.打开共享功能：在虚拟机中点到对应的系统上，然后找到“虚拟机设置”->“选项”->“共享文件夹”，选中总是启动，添加共享目录，点击确定。
        2.安装 Vmware 增加工具（可共享以及复制粘贴）：在虚拟机上安装“Vmware Tools”，安装完成后，系统内的左侧多了 Vmware Tools，点进去，将文件夹下的 VMwareTools-10.1.6-5214329.tar.gz 压缩工具包，拷贝到 Documents 目录下，再使用命令行终端解压 tar -zxvf VMwareTools-10.1.6-5214329.tar.gz，解压完后进入 vmware-tools-distrib 目录，运行 vmware-install.pl，一路 yes 和回车（遇到有 YES 的地方选 yes，其他没有的直接回车，即默认值就可）。
        3.需要进行挂载：安装完成后共享是需要挂载才能看到的，而挂载之前需要安装一个插件 sudo apt-get install open-vm-tools-desktop fuse，之后再输入 sudo vmhgfs-fuse .host:/ /mnt/hgfs/ 挂载命令，之后重启，然后再 cd 到根目录下的 mnt/hgfs/ 就可以看到设置的共享目录里的内容了

      Virtualbox 共享实现：
        1.先配置整个 virtulbox 的设置：打开对应的系统，“控制”->“设置”->“常规”->“高级”，在这里设置虚拟机和主机的双向拷贝实现
        2.在上面同一位置：“共享文件夹”，右边有个 + 符号，点击设置，设置里面的主机共享路径，共享名称，然后自动挂载勾上，挂载点是虚拟机种的系统挂载位置（可设置为 /mnt/share），勾上固定分配，最后确定，这时共享这里就会有刚刚配置的参数了
        3.安装增强工具：虚拟机中的系统，然后“设备”->“安装增强功能”，点击“运行”，输入超级用户密码进行授权，然后开始下载了，知道出现“press return to close this window”按回车，这时就安装成功了。若出现“未能挂载 xxx”
        4.重启后，到挂载位置就可以看到共享的目录了。可能需要管理员权限，后面可以按情况更改权限。
    3. 安装过程中出现 Reading package lists...Done Building dependency tree...：
      原因：这种需要更新下 APT 库
      解决方式：在命令行先输入 sudo rm /var/lib/apt/lists/* -vf，然后输入 sudo apt-get update，之后再输入 sudo apt-get upgrade 即可，之后就可以正常使用 sudo apt-get install xxx 了。
    4. cp: omitting directory './xxx/'：
      原因：因为 xxx 目录下还有目录，所以不能直接使用 cp 命令进行拷贝
      解决方式：-r 选项能实现目录里面还有目录的拷贝，所以是 cp -r ./xxx/ /mnt/xx/
    5. sudo: cd:找不到命令：
      原因：因为 sudo 仅仅对应用程序起作用，而 cd是命令，所以报错。
      解决方式：sudo i，然后再进行 cd 命令，即临时提升到管理员权限
