argparse 模块


ArgumentParser
  1. prog：程序名称
  2. description：帮助文档前文本
  3. usage：描述程序用途
  4. epilog：帮助文档后文本
  5. formatter_class：自定义帮助文档输出格式
  6. prefix_chars：可选参数的前缀字符集合
  7. add_help：添加 -h/--help 选项
  8. parents：解析器对象的列表，它们的参数也应包含在内
  9. allow_abbrev：缩写长选项
  10. fromfile_prefix_chars：从文件中读取其他参数时，标识文件名的前缀字符集合
  11. argument_default：参数的全局默认值
  12. conflict_handler：解决冲突选项的策略
  13. exit_on_error：当错误发生时是否让解析器附带错误信息退出。
add_argument
  1. name or flags：必选，指定参数的形式
  2. default：参数默认值
  3. action：布尔参数
  4. help：帮助信息
  5. metavar：参数在帮助信息中的名称
  6. required：必需参数
  7. type：参数类型
  8. choices：设置参数的范围
  9. nargs：指定参数后面的 value 有多少个
  10. dest：解析出来的 value 放入设置的属性中
  11. const：被一些 action 和 nargs 选择所需求的常数
parse_args
  1. args：要解析的字符串列表
  2. namespace：用于获取属性的对象
add_subparsers
  1. title ：输出帮助的子解析器分组的标题
  2. description：输出帮助中对子解析器的描述
  3. prog：将与子命令帮助一同显示的用法信息，默认为程序名称和子解析器参数之前的任何位置参数。
  4. parser_class：将被用于创建子解析器实例的类，默认为当前解析器类（例如 ArgumentParser）
  5. action：当此参数在命令行中出现时要执行动作的基本类型
  6. dest：将被用于保存子命令名称的属性名；默认为 None 即不保存任何值
  7. required：是否必须要提供子命令，默认为 False (在 3.7 中新增)
  8. help：在输出帮助中的子解析器分组帮助信息，默认为 None
  9. metavar：帮助信息中表示可用子命令的字符串；

ArgumentParser：创建一个解析器

prog：程序的名称（默认：sys.argv[0），可用“%(prog)s”获取此参数。
默认解析器使直接使用sys.argv[0]确定程序名称，如test.py则此名称就是test.py

def funt():
 parser = argparse.ArgumentParser(prog="argparse")
 parser.print_help()

if __name__ == "__main__":
 funt()

formatter_class：通过指定备用格式化类来自定义帮助格式。目前有四类：

 1.class argparse.RawDescriptionHelpFormatter：在正文的描述和展示上给与了更多的控制。对象会将 description 和 epilog 的文字在命令行中自动换行。传此类给 formatter_class= 表示 description 和 epilog 已经被正确的格式化了，不能在命令行中被自动换行。注意：多重的新行（什么信息都没有直接换行）则只会被替换成一行，若想保留多重空白行则可在新行之间加空格。

 2.class argparse.RawTextHelpFormatter：功能和第一个一样

 3.class argparse.ArgumentDefaultsHelpFormatter：自动添加默认值的信息到每一个帮助信息参数中

 4.class argparse.MetavarTypeHelpFormatter：为它的值在每一个参数中使用 type 的参数名当作它的显示名（而不是使用通常的格式 dest )

** formatter_class 举例 ** 例1：
import argparse
decript="""
-----------------
 
 
argparse module.
-----------------
"""
def funt():
 parser = argparse.ArgumentParser(description=decript,epilog=decript,formatter_class=argparse.RawDescriptionHelpFormatter)
 args=parser.parse_args()
 parser.print_help()

if __name__ == "__main__":
 funt()

默认：

add_help=False：


parents：解析器使用一系列相同参数时，单个解析器中参数可直接通过parents=使用相同的参数而不是重复这些参数的定义
（此时要将父解析器 add_help 置为 false，否则会报错有多个 --help 功能）
注意：1.若父解析器使用了特殊的 @，则子解析器要使用相同参数时必须也要指定 此特殊符号；
   2.令父解析器 add_help 为 false，否则父子解析器都有 --help 功能会产生错误)。

** parents 举例 ** import argparse
def funt():
 parser = argparse.ArgumentParser(prefix_chars="@-",add_help=False) #此处必须使add_help为False
 parser.add_argument("@info")
 child=argparse.ArgumentParser(prefix_chars="@-",parents=[parser])
 child.parse_args(['@info',"123"])
 bar_parser = argparse.ArgumentParser(parents=[parser])
 bar_parser.add_argument('--bar')

if __name__ == "__main__":
 funt()

allow_abbrev：若缩写无歧义则允许缩写长选项（默认值：True），若设置为False则长选项必须全部写完整。


fromfile_prefix_chars：当处理一个特别长的参数列表的时候，把它存入一个文件中而不是在命令行打出来会很有意义。如果 fromfile_prefix_chars= 参数提供给 ArgumentParser 构造函数，之后所有类型的字符的参数都会被当成文件处理，并且会被文件包含的参数替代。
  从文件读取的参数在默认情况下必须一个一行（但是可参见 convert_arg_line_to_args()）并且它们被视为与命令行上的原始文件引用参数位于同一位置。
  默认为 None，意味着参数不会被当作文件对待

** fromfile_prefix_chars 举例 ** import argparse

def funt():
 with open('args.txt', 'w') as fp:
  fp.write('-f\nbar')
 parser = argparse.ArgumentParser(fromfile_prefix_chars='@')
 parser.add_argument('-f')
 parser.parse_args(['-f', 'foo', '@args.txt'])#相当于['-f', 'foo', '-f', 'bar']

if __name__ == "__main__":
 funt()

argument_default：一般情况下，参数默认会通过设置一个默认到 add_argument() 或者调用带一组指定键值对的 ArgumentParser.set_defaults() 方法。但是有些时候，为参数指定一个普遍适用的解析器会更有用。这能够通过传输 argument_default= 关键词参数给 ArgumentParser 来完成。


conflict_handler：默认情况下，对象不允许在相同选项字符串下有两种行为，若有则会异常保错，有些时候（例如：使用 parents），重写旧的有相同选项字符串的参数会更有用。为了产生这种行为， 'resolve' 值可以提供给 ArgumentParser 的 conflict_handler= 参数

** conflict_handler 举例 ** import argparse

def funt():
 parser = argparse.ArgumentParser(conflict_handler='resolve')
 parser.add_argument('-f','--f',help="old funtion.")
 parser.add_argument('--f',help="new funtion.") #这里的--f被重写
 parser.print_help()

if __name__ == "__main__":
 funt()

exit_on_error：正常情况下，当你向 ArgumentParser 的 parse_args() 方法传入一个无效的参数列表时，它将会退出并发出错误信息。
如果用户想要手动捕获错误，可通过将 exit_on_error 设为 False 来启用该特性。注意：此为 3.9 新版功能.

** exit_on_error 举例 ** import argparse

def funt():
 parser = argparse.ArgumentParser(exit_on_error=False)
 parser.add_argument('--integers', type=int)
 try:
  parser.parse_args('--integers a'.split())
 except argparse.ArgumentError:
  print('Catching an argumentError')

if __name__ == "__main__":
 funt()

default：参数默认值，指定了在命令行参数未出现时应当使用的值，可通过 %(default)s 来获取默认参数，默认值为 None
对于 nargs 等于 ? 或 * 的位置参数， default 值在没有命令行参数出现时使用。（如：
 parser.add_argument('foo', nargs='?', default=42)
 parser.parse_args(['a'])
 parser.parse_args([])
）
提供 default=argparse.SUPPRESS 导致命令行参数未出现时没有属性被添加，如：

 funt()

action：将命令行参数与动作相关联。这些动作可以做与它们相关联的命令行参数的任何事，尽管大多数动作只是简单的向 parse_args() 返回的对象上添加属性。

 store：存储参数值，

 store_const：存储被const命名参数指定的值

 store_true和store_false：只有真假两种结果 bool 类型，若有指定参数时为真，否则为假

 append：存储一个列表并将每个参数值追加到列表中

 append_const：存储一个列表并将const命名参数指定的值追加到列表中（注意 const 命名参数默认为 None。）

 count：计算一个关键字参数出现的次数，默认为None，除非显示的设为0

 help：打印所有当前解析器中的选项和参数的完整帮助信息，然后退出。默认情况下，一个 help 动作会被自动加入解析器。

 version：打印版本信息并在调用后退出

 extend：存储一个列表，并将每个参数值加入到列表中。

** action 举例 ** import argparse
def funt():
 parser = argparse.ArgumentParser()
parser.add_argument("--abc","-ABC",action='store_true')
 args=parser.parse_args()
 print (args)
if __name__ == "__main__":
 funt()

required：必需参数，若此值为 true，则参数必须要写。若未出现在命令行则会报错。


type：参数类型，不能和 action 参数一起使用。类型有 int、str、float、tuple、dict、open 等。可通过 %(type)s 获取 type 参数。

为方便使用不同类型的文件，argparse 模块提供了 FileType 工厂类，该类接受 mode=, bufsize=, encoding= 和 errors= 等 open() 函数参数。 例如，FileType('w') 可被用来创建一个可写文件。
由于帮助字符串支持 %-formatting，如果你希望在帮助字符串中显示 % 字面值，你必须将其转义为 %%。

const：保存不从命令行中读取但被各种 ArgumentParser 动作需求的常数值。最常用的两例为：
当 add_argument() 通过 action='store_const' 或 action='append_const 调用时。这些动作将 const 值添加到 parse_args() 返回的对象的属性中。在 action 的描述中查看案例。
当 add_argument() 通过选项（例如 -f 或 --foo）调用并且 nargs='?' 时。这会创建一个可以跟随零个或一个命令行参数的选项。当解析命令行时，如果选项后没有参数，则将用 const 代替。在 nargs 描述中查看案例。
对 'store_const' 和 'append_const' 动作， const 命名参数必须给出。对其他动作，默认为 None。


parse_args方法

args：要解析的字符串列表，默认值从 sys.argv 中获取。

对于长选项（名称长度超过一个字符的选项），选项和值也可以作为单个命令行参数传入，使用 = 分隔它们即可。如：
 parser.parse_args(['--foo=FOO'])

对于短选项（长度只有一个字符的选项），选项和它的值可以拼接在一起。如：
 parser.add_argument('-x')
 parser.parse_args(['-xX'])

有些短选项可以使用单个 - 前缀来进行合并，如果仅有最后一个选项（或没有任何选项）需要值的话，如：
 parser.add_argument('-x', action='store_true')
 parser.add_argument('-y', action='store_true')
 parser.add_argument('-z')
 parser.parse_args(['-xyzZ'])

namespace：用于获取属性的对象。默认值为 None。

** namespace 举例 ** import argparse
class C:
 pass
def funt():
 c=C()
 parser = argparse.ArgumentParser()
 parser.add_argument("--abc","-ABC")
 parser.parse_args(args=['--abc', 'BAR'], namespace=c)
 print (c.abc)

if __name__ == "__main__":
 funt()
