  1. git init：创建本地仓库（在此目录下会创建隐藏项目.git目录）
  2. git clone：将远程仓库项目检出到本地仓库。通常url最后一个 / 之后的项目名称，若想改名称则在该命令后加上要改的名称
  3. git config --global user.name或user.email： 配置用户名和邮箱
  4. git config -l：查看配置
  5. git status：获取当前状态，如是否已加入到本地缓存。
  6. git add xxx：将文件添加到缓存。如git add ./，即加入当前目录下全部已改内容
  7. git commit -m "xxx"：将文件提交到仓库，-m后面输入的是本次提交的说明，如git commit -m "增加版本V1.1"
  8. git diff：对比修改增减等不同的部分
  9. git push origin xxx：提交到远程仓库（这里会跳出一个提示框要求输入用户名和密码，若远程无此分支则会创建一个新分支）
  10. git push origin --delete xxx：删除远程分支，需要输入用户名和密码。
  11. git pull origin xxx：检出远程仓库
  12. git branch：查看分支（不带选项为本地分支，-r为远程分支，-a为全部分支）
  13. git branch -m oldxx newxxx：重新命名分支名称
  14. git branch -d xxx：删除本地分支，此分支未被改动过。此时应该切换到其他分支或主分支上进行操作。若想强制删除则将 -d 改成 -D 即可。
  15. git log：查看本地提交的log日志
  16. git reset --hard xxx：返回到前面已经提交过的一版中
  17. git reset --soft HEAD^：若不小心提交了错误的或想本地仓库撤销，此时不会影响已经改动的代码
  18. git checkout xxx：切换分支，使用选项 -b 即创建分支并切换到此分支上去
  19. git clean -d -fx：为删除一些没有git add的文件，若切换时切换不过去，可以使用这个命令
  20. git show-branch -a --color=always：有若干短划线”-”分隔，几个分支就会有几个短划线。其中短划线分割两部分，上部分为层次缩进的分支列表，下半部分为commit列表。上部分*标识当前分支，！标识其他分支，不同分支颜色不同。下部分commit列表的*和+,*标识这列中当前分支由此commit，而+标识这列中的非当前分支由此commit。颜色用于区分分支
  21. git show -a --color=always：若commit之后还想再看下提交前后之间的有哪些地方改变了，则可以使用此命令查看
  22. git stash：将文件放入暂存区
  23. git stash list：列出所有stash
  24. git stash apply：恢复暂存区内容
  25. git stash drop：删除暂存区
  26. git tag xxx：给当前 log 打上 tag 标签，如发布版本可打上 tag 标签，这样 git 上就能看出哪个 log 是发布版本
  27. git tag xxx logid：给 log 为 logid 的位置打上 tag 为 xxx 的标签
  28. git reflog：查看本地记录
  30. 错误解决：
    • error: The following untracked working tree files would be overwritten by checkout:
      一般是改了代码，可以使用git status查看下，此时可以进行代码提交，然后再切换分支，或者并不想要这个已改的代码，则可以使用git reset --hard && git clean -dxf恢复原代码，再切换。
    • error: failed to push some refs to 'https://xxx/xx.git'
      主要原因使远程库和本地库不一致导致的，可使用git pull --rebase origin master 把远程库中的更新合并到（pull=fetch+merge）本地库中，–-rebase的作用是取消掉本地库中刚刚的commit，并把他们接到更新后的版本库之中。再执行git push -u origin master将代码上传
    • 错误 xxx|REBASE 1/234：
      原因：在本分支上进行代码 add 和 commit 之后，使用了 git push --rebase origin xxx，push 后有冲突，再使用 git pull --rebase origin master 拉错了分支，导致此问题出现。
      解决方式：git rebase --abort，然后使用git pull origin xxx拉取最新分支，若出现 xxx|MERGING 冲突，则使用git diff获取下冲突文件的区别，里面有提示当前和冲突的log，此时再使用git reset --hard xx恢复到冲突的分支，再用git pull origin xxx分支，得到最新的代码
    • The requested URL returned error: 502
    • 创建新版本库：
      touch README.md
      git init
      git add README.md
      git commit -m "first commit"
      git remote add origin ssh://ip:port/xxx/xx.git
      git push -u origin master
    • error: Cannot delete branch 'master' checked out at 'x://xx//xxx'
      原因：在当前分支删除当前分支时的错误
      解决方式：想删除 master 需要跳转到其他分支，然后再使用 branch -D master 删除 master 分支
      若只有当前一个分支，只需删除文件夹内所有的文件，包括 .git 文件，再使用 git init、git clone 重新拉取远程主分支 master 即可

小技巧1：若本地可执行的git在远程中始终进入不了git版本管理时，这时可以打开git-bash.exe。然后在远程上找到本地需要使用到的目录，使用cd \\c:\x\xx\xxx（这里需要将反斜杠改成斜杆）。即可进入

小技巧2：若git-bash只能在指定的目录下运行git命令，则打开后先输入 export PATH=/mingw64/bin:$PATH 。之后就能正常使用git了。
